The Missing Semester of Your CS Education

1. 课程概览与Shell

Shell 是一种命令行解释器，用于与操作系统进行交互。它允许用户通过输入命令来执行各种操作，比如文件管理、进程控制、程序执行等。在类 Unix 系统中，最常用的 shell 包括 Bash、Zsh 和 Ksh 等。

Shell 脚本是一系列 shell 命令的集合，可以用来自动化任务、简化复杂操作。虽然 shell 脚本的语法可能看起来简单，但它们能够处理很多复杂的功能。

我个人对 shell 语言进行了简单的学习，并发现实际使用中常用的 shell 语句并不多。对于较为复杂的功能需求，通常可以通过查阅资料来解决

  1. echo: 用于输出文本或变量的值到终端。
  2. ls: 列出目录中的文件和子目录。
  3. cd: 切换当前工作目录。
  4. pwd: 显示当前工作目录的路径。
  5. cp: 复制文件或目录。
  6. mv: 移动文件或目录，或者重命名。
  7. rm: 删除文件或目录。
  8. grep: 在文件中搜索指定的模式。
  9. find: 查找符合条件的文件或目录。
  10. chmod: 修改文件或目录的权限。

2. Shell工具和脚本

到目前为止，我们已经学习了如何在 Shell 中执行命令，并使用管道将命令组合起来。然而，很多时候我们需要执行一系列操作，并使用条件或循环这样的控制流。

简要记录

  • shell 脚本支持 if、case、while 和 for 等控制流关键字，并支持函数，函数可以接受参数并基于这些参数进行操作。
  • 特殊变量：
    • $0 - 脚本名
    • $1 到 $9 - 脚本的参数。$1 是第一个参数，依此类推。
    • $@ - 所有参数
    • $# - 参数个数
    • $? - 前一个命令的返回值
    • $$ - 当前脚本的进程识别码
    • !! - 完整的上一条命令，包括参数。例如，当命令因为权限不足而失败时，可以用 sudo !! 再尝试一次。
    • $_ - 上一条命令的最后一个参数。在交互式 Shell 中，可以通过按 Esc 后输入 . 来获取这个值
  • shell 命令通常使用 STDOUT 返回输出值，使用 STDERR 返回错误和错误码。返回值 0 表示正常执行，其他非 0 的返回值表示有错误发生。
  • 使用 $( CMD ) 可以将 CMD 的输出替换掉 $( CMD )
